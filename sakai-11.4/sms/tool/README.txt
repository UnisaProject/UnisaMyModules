ABOUT THE SMS RSF WINDOWS
=========================
The ui is currently in a separate Eclipse project. It will be merged into the sms tool project later on. 
They are currently stand-alone RSF windows that run in jetty. Jetty is used during the
development process for its startup speed. Sakai stylesheets has been attached. 

STARTING UP JETTY
=================
* Make sure the sms_ui project is built using: mvn clean install
* To change the default port 8080, edit the <port> tag in pom.xml
* Start up jetty using: mvn jetty:run-war

SMS TEST WINDOW
===============
http://localhost:8080/sms_ui/faces/sms_test
Note: Not in use anymore.
The sms test window is used to send a single message to the gateway. It is used to test the smpp service. This window will 
later be changed to send messages to a Sakai group. When a message is send from the window, the jetty log and 
the simulator log should be monitored to see the response. The smpp service will report the gateway message id in 
the debug panel. 

SMS HELPER WINDOW
=================
http://localhost:8080/sms_ui/faces/helper
This is currently a stand-alone rsf window and could be change into a helper window if required. It demonstrate how to implement SMS 
in a existing Sakai tool. The sms helper window is used to insert a sms task into the task table. 
The scheduler will then pick up the task and process it.  
the debug panel. 

SMS CONFIG WINDOW
=================
http://localhost:8080/sms_ui/faces/SmsSiteConfig
There is a config window for the specific site (currently hard-code) and for the system.
 
SMS ACCOUNT ADMIN WINDOW
========================
http://localhost:8080/sms_ui/faces/billing_admin
This will list all sms accounts in the current Sakai site. Accounts can be added and changed. 
Sakai permissions will be added in delivery 7. This interface will also handle system-wide sms accounts and accounts per user

SMS TASK WINDOW
===============
http://localhost:8080/sms_ui/faces/TaskList
List all tasks for the current site, or system wide. Tasks can be sorted on all columns and the search 
filter can be applied to quickly find records. A new task can also be created from here.

SMS MESSAGE WINDOW
==================
http://localhost:8080/sms_ui/faces/MessageLog
Show all messages with statuses. Columns can be sorted and messages can be located using the search filter.

SMS TRANSACTION WINDOW
======================
http://localhost:8080/sms_ui/faces/TransactionLog
Show all sms transactions generated by the sms billing service. Columns can be sorted and transactions can be located 
using the search filter.
